# 컨텍스트 기반 질의 정확도 향상을 위한 전처리 가이드

## 목적

이 문서는 한국은행 보수규정처럼 조문, 표, 계산 규칙이 함께 존재하는 문서를 LLM 컨텍스트로 직접 넣어 질의응답할 때, 실수를 줄이고 답변 정확도를 높이기 위한 전처리 원칙을 정리한다.

이번 저장소의 비교 결과에서 확인된 핵심은 다음과 같다.

- 구조화 그래프 질의는 이미 템플릿화된 조회에는 강하다.
- 문서 해석과 외부 입력값 계산이 섞인 질문에서는 컨텍스트 기반 접근이 더 자연스럽다.
- 다만 문서를 그대로 넣는 것만으로는 충분하지 않고, LLM이 틀리기 쉬운 지점을 전처리로 먼저 제거해야 한다.

즉, 좋은 전처리의 목적은 문서를 요약하는 것이 아니라 LLM이 선택, 매핑, 계산, 제외 조건 판단을 틀리지 않게 만드는 것이다.

## 전처리의 핵심 원칙

### 1. 규정 원문을 그대로 두지 말고 질의 단위로 재구성한다

문서형 질의는 보통 아래 네 단계를 거친다.

1. 질문에서 핵심 개체와 조건을 해석한다.
2. 어떤 규칙 또는 표를 써야 하는지 선택한다.
3. 필요한 값을 찾는다.
4. 계산 또는 판정을 수행한다.

원문 PDF는 이 네 단계를 직접 지원하지 않는다. 따라서 전처리 문서는 다음 관점으로 재구성해야 한다.

- 조문: 조건 → 결과 형태로 재서술
- 표: 키, 용도, 반환값이 드러나게 정리
- 계산 규칙: 공식 블록으로 명시
- 예외 규정: 별도 섹션으로 분리
- 고위험 질문: QA 형태로 정답 예시 수록

### 2. 표마다 적용 조건과 금지 조건을 함께 적는다

LLM은 값을 못 읽어서 틀리기보다, 틀린 표를 선택해서 틀리는 경우가 많다.

아래 두 정보가 반드시 필요하다.

- 이 표를 언제 써야 하는가
- 이 표를 언제 쓰면 안 되는가

예시:

```md
### 연봉 차등액표
- 키: 직급 + 평가등급
- 용도: 연봉제 본봉 조정 계산
- 반환값: 연간 차등액
- 제외: 직책급 계산, 상여금 계산, 임금피크제 계산에는 직접 사용하지 않음
```

### 3. 계산 규칙은 조문 설명이 아니라 공식으로 적는다

조문 요약만 있으면 LLM이 중간 규칙을 자의적으로 보완한다. 계산이 필요한 항목은 반드시 입력값, 공식, 주의사항을 구조화해서 적는다.

예시:

```md
## 계산 규칙

### 연봉제본봉 조정
- 적용 조건: 연봉제 적용대상 직원
- 입력값:
  - 현재 연봉제 본봉
  - 직급
  - 평가등급
- 공식:
  - 조정 후 연봉제 본봉 = 현재 연봉제 본봉 + 해당 직급/평가등급의 연봉 차등액
- 주의:
  - 직책급은 연봉제 본봉 조정 공식에 자동 합산하지 않음
  - 질문에 연봉 상한이 포함되면 상한 적용 여부를 별도로 판단
```

### 4. 동의어와 표현 변형을 정규화한다

질문은 표의 정식 명칭과 다르게 들어온다. 따라서 용어 정규화 사전이 필요하다.

예시:

```md
## 용어 정규화

- G5 = 종합기획직원 5급 = 종합기획 5급
- G3 = 종합기획직원 3급 = 종합기획 3급
- 직급평점 = 평가등급
- 본봉 조정 = 연봉제본봉 조정
- 국외본봉 = 해외 기본급 = 주재 본봉
```

### 5. 예외와 적용 제외 규정을 별도 섹션으로 뺀다

LLM은 없는 값을 만드는 것보다, 적용 제외 규정을 놓쳐서 틀리는 경우가 많다.

예시:

```md
## 적용 제외 및 주의사항

- 기한부 고용계약자는 제2장 보수 및 제3장 상여금 규정을 적용하지 않음
- 연봉 차등액표는 1급~3급에만 적용됨
- 질문에 현재 금액이 주어지면 표의 본봉 금액이 아니라 입력 금액을 기준으로 계산함
- 직책급은 기본급의 구성요소이지만, 연봉제본봉 조정 공식에 자동 합산하지 않음
```

### 6. 정답 예시를 숨기지 말고 QA 블록으로 제공한다

반복적으로 등장하는 고위험 질문은 QA 블록으로 직접 수록하는 편이 정확도에 유리하다.

예시:

```md
## 질의 예시

Q. G5 직원의 초봉은?
A. 종합기획직원 5급의 초임호봉은 11호봉이며, 5급 11호봉 본봉은 1,554,000원이다.

Q. 현재 연봉제 본봉이 70,000,000원이고 3급 EE이면 조정 후 얼마인가?
A. 3급 EE 차등액은 2,016,000원이므로 72,016,000원이다.
```

## 권장 전처리 구조

하나의 전처리 문서는 아래 구조를 권장한다.

```md
# 문서 제목

## 문서 사용 원칙
## 용어 정규화
## 질의 해석 메모
## 적용 제외 및 주의사항
## 계산 규칙
## 주요 조문
## 표 메타데이터
## 실제 표 데이터
## 질의 예시
```

### 섹션별 역할

- 문서 사용 원칙: 답변 범위와 제한 정의
- 용어 정규화: 질문 표현을 내부 용어로 통일
- 질의 해석 메모: 자주 틀리는 해석 포인트 명시
- 적용 제외 및 주의사항: 예외 처리 강화
- 계산 규칙: 공식 기반 답변 유도
- 주요 조문: 판정 근거 제공
- 표 메타데이터: 어떤 표를 써야 하는지 안내
- 실제 표 데이터: 수치 근거 제공
- 질의 예시: 고빈도 질문 정답 패턴 제공

## 표 전처리 체크리스트

모든 표는 아래 항목을 충족하는 것이 좋다.

- 표 이름이 질문 표현과 맞닿아 있는가
- 키 컬럼이 명확한가
- 반환값 컬럼이 명확한가
- 단위가 분리되어 있는가
- 기간 단위가 명시되어 있는가
- 적용 대상과 제외 대상이 적혀 있는가
- 동의어가 별도 사전에 등록되어 있는가
- 질문에 바로 재사용할 수 있는 예시가 있는가

좋은 예:

```md
### 국외본봉표
- 키: 국가 + 직급
- 반환값: 월 국외본봉, 통화
- 용도: 해외 주재 직원 기본급 질의
- 제외: 조정수당, 세금, 환율 계산에는 직접 사용하지 않음

| 국가 | 직급 | 월 국외본봉 | 통화 |
| --- | --- | ---: | --- |
| 미국 | 1급 | 10,780 | USD |
| 미국 | 2급 | 9,760 | USD |
```

나쁜 예:

- 표 제목이 원문 그대로라 질문과 연결되지 않음
- 열 이름이 축약돼 의미가 불명확함
- 월/연 단위가 섞여 있음
- 적용 범위가 없음

## 계산형 질문 정확도 향상 가이드

계산형 질문은 아래 네 가지를 반드시 전처리해야 한다.

### 1. 입력값과 조회값을 구분한다

질문에 이미 숫자가 들어오면 그 숫자는 표에서 다시 찾는 값이 아니라 계산 입력값이다.

예시:

- 질문: 현재 연봉제 본봉이 70,000,000원이면 조정 후 얼마인가?
- 해석:
  - 입력값: 70,000,000원
  - 조회값: 3급 EE 차등액 2,016,000원
  - 계산: 입력값 + 조회값

### 2. 자동 합산하면 안 되는 항목을 명시한다

LLM은 직책급, 상여금, 차등액을 한꺼번에 더하려는 경향이 있다. 따라서 비합산 항목을 별도 문장으로 강하게 박아두는 것이 좋다.

예시:

- 연봉제본봉 조정 계산에는 직책급을 자동 합산하지 않음
- 상여금은 별도 지급 항목이며 본봉 조정 공식에 포함하지 않음

### 3. 계산 예시를 같이 준다

공식만 있는 것보다 실제 숫자 예시가 있는 편이 오류가 적다.

예시:

```md
예시:
- 현재 연봉제 본봉: 70,000,000원
- 직급: 3급
- 평가등급: EE
- 차등액: 2,016,000원
- 조정 후 연봉제 본봉: 72,016,000원
```

### 4. 계산 결과 형식을 고정한다

답변 형식을 전처리 문서나 프롬프트에서 고정하면 설명이 흔들려도 결과가 덜 틀린다.

예시:

- 답변 형식: 적용 규칙 → 사용 값 → 계산식 → 최종 금액

## 질의 유형별 추천 전처리

### 규정 해석형 질문

예: 기한부 고용계약자는 상여금을 받을 수 있어?

필수 전처리:

- 관련 조문 요약
- 적용 제외 규정
- 결론형 문장

권장 답변 근거 구조:

- 조건: 기한부 고용계약자
- 규정: 제14조
- 결론: 상여금을 받을 수 없다

### 단순 조회형 질문

예: G5 직원의 초봉은?

필수 전처리:

- 초임호봉표
- 본봉표
- 용어 정규화
- 직접 매핑 예시

### 비교형 질문

예: 미국과 일본의 1급 국외본봉 비교

필수 전처리:

- 동일 단위 표 유지
- 국가명 표기 통일
- 통화 정보 명시
- 비교 시 환산하지 않는다는 규칙 명시

### 계산형 질문

예: 현재 연봉제 본봉이 70,000,000원이면 조정 후 얼마인가?

필수 전처리:

- 계산 공식
- 입력값/조회값 구분
- 비합산 규칙
- 계산 예시

## 전처리 문서 품질 점검 질문

전처리 후 아래 질문에 모두 예라고 답할 수 있어야 한다.

1. 질문 표현이 달라도 같은 항목으로 매핑되는가
2. 어떤 표를 써야 하는지 문서만 보고 알 수 있는가
3. 어떤 값을 더하면 안 되는지 문서만 보고 알 수 있는가
4. 계산 입력값과 조회값을 구분할 수 있는가
5. 적용 제외 규정이 별도 섹션으로 분리돼 있는가
6. 고빈도 질문에 대한 정답 예시가 있는가
7. 숫자의 단위와 기간 단위가 명확한가

## 이 저장소에 바로 적용할 우선순위

현재 저장소 기준으로는 아래 순서가 가장 효과적이다.

1. `regulation_context.md`에 `용어 정규화`, `적용 제외 및 주의사항`, `계산 규칙` 섹션 강화
2. 각 표 앞에 `키`, `용도`, `반환값`, `제외` 메타데이터 추가
3. 연봉제본봉, 상여금, 임금피크제, 국외본봉 관련 고위험 질의 예시 추가
4. 질문 유형별로 반드시 포함해야 할 섹션을 선택하도록 검색 로직 보강

## 결론

컨텍스트 기반 질의응답의 품질은 문서를 많이 넣느냐보다, 문서가 LLM의 의사결정을 얼마나 강하게 제한하느냐에 달려 있다.

가장 효과적인 전처리는 다음 다섯 가지다.

- 용어 정규화
- 적용 제외 규정 분리
- 계산 규칙의 공식화
- 표 메타데이터 부여
- 고위험 질문의 QA 예시화

이 다섯 가지가 갖춰지면, LLM은 원문을 읽는 모델이 아니라 잘 정리된 규칙집을 실행하는 모델에 가까워진다.
